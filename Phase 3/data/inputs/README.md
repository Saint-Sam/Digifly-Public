# Phase 3 Inputs

Use this folder for all Phase 3 input references.

## Current subfolders

- `mappings`
  - Copied mapping files from legacy Phase 3 root.
- `phase2_runs`
  - Place run-specific pointers or copied outputs from Phase 2 runs.

## Suggested convention for `phase2_runs`

Create one folder per run ID (for example `run_debug_full`) and keep:

- `spike_times.csv`
- `records.csv` (optional but recommended for exact timebase)
- Any run metadata needed to reproduce the bridge output.
